    粘液科技扩展玩法指南¶

    本页介绍服务器的粘液科技的三项扩展：生态能源、蓬松机器与网络化存储。
    能源 EcoPower 生态能源
    生产 FluffyMachines 蓬松机器
    仓储 Networks 网络化存储
    适合阶段 粘液科技中期至后期
    主要目标 建设完整科技工业体系

    重要说明

    所有具体配方、耗电量、机器速度、多方块结构和可到达天体， 均以服务器当前版本的粘液科技指南为准。外部 Wiki 主要用于理解机制， 部分旧页面可能与服务器实际版本存在差异。

    能源 · 生态能源｜EcoPower¶

    EcoPower 用于扩充粘液科技的发电体系。它让玩家不再只依赖燃烧燃料或单一太阳能设备， 而是根据基地环境建设风力、蒸汽和天气相关能源设施。

    核心玩法¶

    风力发电¶

    利用风力设备持续产生电力，适合建设在开阔区域或作为基地的长期基础电源。

    蒸汽发电¶

    通过热能与蒸汽体系发电，适合与工业区统一规划，形成较稳定的集中供电设施。

    雷电能源¶

    利用雷雨天气收集能量。产能具有环境条件，适合与其他电源搭配，而不是作为唯一电源。

    混合电网¶

    将多种发电机接入电容与能源网络，在天气、时间或负载变化时保持工厂持续运行。

    玩家入门步骤¶

    1. 确认用电规模。 先计算当前机器数量，避免只造发电机却没有足够储电能力。
    2. 选择合适能源。 开阔基地可优先考虑风力；工业基地可考虑蒸汽；雷电设备作为补充。
    3. 配置电容。 发电量会随条件变化，电容可以储存富余电力并缓冲短时缺电。
    4. 建立分区电网。 建议将农业、矿物加工、仓库和网络存储分成不同用电区域。

    推荐用途

    为蓬松机器的自动生产线以及 Networks 网络节点提供长期电力。

    资料入口

    • 英文项目仓库：EcoPower GitHub

    • 鬼斩中文 Wiki：目前未发现独立的 EcoPower 专题页，请优先查看游戏内指南。

    生产 · 蓬松机器｜FluffyMachines¶

    FluffyMachines 是一套偏实用型的自动化扩展。它把原本需要玩家反复手动完成的工作台、 魔法工作台、盔甲锻造炉、远古祭坛和台锯操作，转换为可以接入电力和物流的自动生产流程。

    主要机器与用途¶

    类型 用途 典型应用
    自动工作台 自动执行原版合成配方 木板、木棍、箱子、活塞等基础物资流水线
    自动魔法工作台 自动执行魔法工作台配方 批量生产符文、护身符及魔法材料
    自动盔甲锻造炉 自动执行盔甲锻造炉配方 批量生产粘液科技护甲与相关部件
    自动远古祭坛 将祭坛配方改为机器化生产 减少反复摆放祭坛材料的操作
    自动台锯 自动执行木材加工配方 建筑材料批量生产
    粉尘制造与回收 处理筛矿、粉尘与矿物中间产物 矿物加工生产线与废料回收
    高级充电与便携充电 为电力工具和装备补充电力 矿场、远征和工厂维护

    如何搭建第一条自动生产线¶

    发展路线： 原料箱 → 货运输出节点 → 自动机器 → 货运输入节点 → 成品箱或网络仓库

    1. 先选择单一成品。 建议从木板、木棍或常用部件开始，不要一开始就建设复杂总线。
    2. 为机器持续供电。 机器必须接入能源网络，电力不足会导致生产中断。
    3. 设置原料与成品方向。 使用粘液科技本体的货运节点输送物品。
    4. 逐步串联。 一台机器的成品可以成为下一台机器的原料，最终形成完整流水线。

    其他便利内容¶

    • 洒水器和浇水壶：加快农作物或植物生长。
    • 镰刀：一次处理多株作物。
    • 搬运器：搬运保留内部物品的容器。
    • 末影箱节点：将末影箱接入物品运输流程。
    • 传送板与替代电梯板：用于基地内部短距离移动。
    • 扳手、多功能工具和强化采集工具：提高工厂维护与资源采集效率。

    使用提醒

    大型货运网络和大量持续运行的机器会增加服务器负载。建议把生产线分区建设， 避免一个货运管理器连接过多设备，也不要让无用产物无限堆积。

    • 英文项目仓库：FluffyMachines GitHub
    • 英文官方说明：Cargo Management
    • 鬼斩中文 Wiki：蓬松机器

    仓储 · 网络化存储｜Networks¶

    Networks 提供类似模组化科技中的中央仓储系统。玩家不必逐个翻找大量箱子， 而是通过统一网络终端访问存储空间，并利用导入、导出节点让机器自动调用材料。

    网络由什么组成¶

    网络核心与控制部件¶

    维持网络运行并连接其他节点。不同版本的具体名称和连接限制以游戏内指南为准。

    存储单元¶

    真正保存物品的部分。玩家可逐步增加容量，而不需要继续铺设成排普通箱子。

    访问终端¶

    集中查看、搜索、存入和取出网络内物品，是日常使用的主要界面。

    导入与导出节点¶

    将外部容器的物品送入网络，或按需求从网络取出特定物品供应机器。

    基础搭建流程¶

    1. 建立网络核心。 在基地仓库中心预留足够空间，避免后期扩建困难。
    2. 接入存储单元。 先存放常见原料和成品，再逐步扩容。
    3. 放置访问终端。 确认可以正常搜索、存入和取出物品。
    4. 配置导入节点。 将农场、矿场和工厂成品自动送入网络。
    5. 配置导出节点。 让自动机器从网络持续获得指定原料。

    与货运系统的区别¶

    系统 主要职责 简单理解
    Networks 集中存储、搜索、调用和分配物品 中央仓库
    Slimefun 货运系统 在箱子和机器之间搬运物品 传送带与物流线路

    发展路线： 网络仓库 → 导出指定原料 → 自动机器加工 → 导入成品 → 重新存入网络

    适合存放的物品

    粉尘、合金、机器零件、导线、电路板、农作物、建筑材料以及其他数量较大的常用资源。

    资料入口

    • 英文项目仓库：Networks GitHub
    • 英文官方文档：Networks Documentation
    • 鬼斩中文 Wiki：Networks

    推荐发展路线¶

    阶段 主要目标 推荐内容
    第一阶段 建立稳定电力 生态能源、基础电容、分区电网
    第二阶段 减少重复手工操作 自动工作台、自动魔法工作台、基础货运
    第三阶段 形成完整生产线 自动远古祭坛、粉尘加工、成品回收
    第四阶段 建设中央仓库 Networks 核心、存储单元、访问终端
    第五阶段 打通仓储与生产 自动导入、自动导出、按需供料

    常见问题¶

    这些扩展需要按固定顺序游玩吗？

    不强制。玩家可以单独研究某一个附加，但按照“能源—生产—仓储”的路线发展会更顺畅。

    货运系统属于蓬松机器吗？

    不属于。货运系统是 Slimefun4 本体提供的物品运输系统。蓬松机器只是让自己的许多机器能够接入这套系统。

    Networks 能完全代替货运系统吗？

    不能简单理解为完全替代。Networks 更偏中央存储和按需调用；货运系统更偏机器与容器之间的物流。两者组合使用效果最好。

    为什么外部 Wiki 的名称和游戏内名称不同？

    不同汉化版本、构建版本和服务器配置可能采用不同译名。查找配方和机器时，应优先使用服务器内粘液科技指南。

    机器为什么突然停止工作？

    常见原因包括电力不足、输出槽堵塞、货运方向设置错误、网络超出连接范围、区块未加载或原料不符合配方。
